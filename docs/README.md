# Docs Index

该目录承载仓库的规则库与知识库，避免顶层 `AGENTS.md` 过度膨胀。

## Rules

- `docs/rules/development-guide.md`
  - 开发流程、架构边界、代码规范、验证矩阵和提交前检查。

## Knowledge Base

- `docs/knowledge-base/project-overview.md`
  - 项目目标、技术栈、目录结构、数据存储、API 与运行方式。
- `docs/experience.md`
  - 开发过程中的经验记录、踩坑背景、用户偏好与后续维护提醒。

## Reading Order

1. 先读 `docs/rules/development-guide.md`，了解本仓库的默认工作方式。
2. 对项目陌生或需要系统上下文时读 `docs/knowledge-base/project-overview.md`。
3. 处理已反复调整过的体验、架构或验证问题时，读 `docs/experience.md` 了解历史经验。
4. 修改结构、脚本、数据契约或 API 时，同步更新相关文档和根目录 `README.md`。
